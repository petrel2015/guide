G1 垃圾回收器的回收过程可以分为几个阶段。
- 主要包括初始标记、并发标记、最终标记和筛选回收。
- 首先是初始标记阶段，这个阶段会标记从 GC Roots 可达的对象，同时标记年轻代中的对象。这个阶段会发生短暂的
停顿，但时间很短。
- 接着是并发标记阶段，G1 会在后台线程中遍历整个堆，标记所有存活对象。这是一个并发过程，不会影响应用线程的执行。
- 然后是最终标记阶段，它会完成并发标记过程中遗漏的部分，比如处理 SATB（Snapshot-At-The-Beginning）队列中的对象变更。这一阶段也会有短暂停顿。 
- 最后是筛选回收阶段，G1 会根据各个 Region 的回收价值进行评估，选择回收收益最高的 Region 进行回收。
- 这一阶段可以是并发的，也可以是停顿式的，具体取决于回收的目标。
- 整个过程的设计目标是尽量减少全堆停顿，并通过预测停顿时间来满足应用的响应需求。
- G1 的这种按 Region 回收、按价值排序的方式，使得它在大堆内存场景下非常高效。
