主要由类加载子系统、运行时数据区、执行引擎和本地接
口构成。

## 类加载器：
类加载器的过程就是如何把 class 文件加载至元空间 metaspace(方法区实现)的过程。类加载器遵循双亲委派原则，加载类时会优先将请求委托给其父加载器，只有当父加载器无法加载时才尝试自己加载，以保证核心类的安全性和统一性。

## 运行时数据区：
分成 2 类，分为线程私有区和线程共享区。线程私有区包括程序计数器（保存下一条执行的字节码地址）、虚拟机栈（栈帧，执行方法调用）、本地方法栈（支持本地方法调用）；线程共享区主要是堆（存放对象实例）和方法区（存放类信息、常量、静态变量，是 jvm 规范）另外值得一提的是，直接内存不是运行时数据区，但是 java 运行过程中也会用到。

## 本地接口：
通过 JNI 提供 Java 与本地代码交互的能力，支持调用底层系统功能和第三方库，扩展了 JVM 的功能边界。

## 执行引擎：
负责将 JVM 字节码解释或编译成机器码并执行。还有垃圾回收器。
