- 我主要使用两类调试工具：JDK 自带工具和 Arthas。
## JDK 工具方面，我常用：
- jstack：查看线程堆栈，用于分析死锁或性能瓶颈；
- jmap：导出堆快照，辅助分析对象分布和内存泄漏；
- jstat：实时监控 GC 状态，不过我们一般还是通过 jvm 参数打开 gc 日志，去观察历史 gc 日志更有价值。 
## 更常用的是 Arthas，它是阿里开源的一款强大的 Java 诊断工具
- 支持热加载、无侵入调试。我常用的命令包括：
- watch：观察方法的入参和返回值；
- trace：分析方法内部的调用路径和耗时；
- stack：查看当前方法的调用链；
- sc 和 sm：分别查看 JVM 加载的类和方法信息。
- 此外，vmtool 是 Arthas 中功能最强但也最复杂的模块之一，它可以直接查询 JVM 中的对象，并调用其方法，适合排查一些复杂的业务逻辑问题。
- 这些工具在我排查过如线程死锁、内存泄漏、接口响应慢等线上问题时都发挥了很大作用。
