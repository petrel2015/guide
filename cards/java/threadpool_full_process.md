---
question: "向 Java 线程池提交一个任务后，会发生什么？"
tags: [ "java", "并发", "线程池" ]
---

- **第一步：判断核心线程数 (corePoolSize)**。如果当前运行的线程数少于核心线程数，则立即创建一个新线程来执行该任务，即使此时有其他核心线程处于空闲状态。
- **第二步：尝试进入阻塞队列 (workQueue)**。如果核心线程已满，线程池会尝试将任务放入阻塞队列。如果队列未满，任务在此等待，直到有核心线程空闲并将其取出。
- **第三步：判断最大线程数 (maximumPoolSize)**。如果阻塞队列也满了，且当前运行线程数小于最大线程数，则线程池会再创建一个非核心线程来执行该任务。
- **第四步：执行拒绝策略 (RejectedExecutionHandler)**。如果线程数已达到最大值，且队列也已填满，此时线程池将根据拒绝策略（如抛异常、丢弃任务或调用者运行）来处理新任务。
- **后续处理：线程存活与销毁 (keepAliveTime)**。当非核心线程空闲时间超过设定值后，它们会被自动销毁，直到线程数回落到核心线程数。
- **总结流程**：核心线程满 -> 队列满 -> 最大线程满 -> 触发拒绝。
