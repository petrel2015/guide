question: "java线程如何创建"

在 Java 中，创建线程主要有三种方式：
1、继承 Thread 类，我们可以定义一个类继承 Thread，并重写它的 run 方法，然后调用 start 方法来启动线程。这种方式简单，但不够灵活，因为 Java 是单继承的，如果类已经继承了其他父类，就不能再继承 Thread。
2、实现 Runnable 接口，我们把任务逻辑写在 run 方法里，然后将这个任务对象传给一个 Thread 实例，再调用start 方法。这种方式更灵活，也更符合面向接口编程的设计原则。
3、实现 Callable 接口，它和 Runnable 类似，但 call 方法可以有返回值，并且支持抛出异常。通常我们会把 Callable 任务提交给 ExecutorService，通过 submit 方法返回一个 Future 对象，用于获取执行结果。

- 虽然这三种方式都可以创建线程，但在实际开发中，我们更推荐使用线程池来管理线程，特别是在高并发或频繁创建线程的场景下。
- 线程池可以复用线程资源，避免频繁创建和销毁线程带来的性能开销，同时也能更好地控制并发数量，提升系统稳定性。
- Java 提供了一个工具类叫 Executors，它提供了几种常用的线程池工厂方法，比如 newFixedThreadPool 创建一个固定大小的线程池。
- 这些方法使用方便，但在生产环境中，我们更多使用 **ThreadPoolExecutor** 来手动创建线程池，因为它可以更精细地控制线程池的行为。
- ThreadPoolExecutor 的构造方法中关键的参数是：核心线程数；最大线程数；任务队列，用于保存等待执行的任务；拒绝策略：当线程池和队列都满了时，决定如何处理新任务。
- 任务队列可以是无界队列，比如LinkedBlockingQueue，也可以是有界队列，比如 ArrayBlockingQueue，甚至可以是带优级的队列。
- 拒绝策略有四种：抛出异常、由提交线程执行、丢弃新任务、或丢弃最旧任务。
- 通过这些参数的组合，我们可以根据业务需求灵活配置线程池，从而实现高效、稳定的并发处理。
