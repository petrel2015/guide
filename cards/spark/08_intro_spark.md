Spark 的目标是通过分布式计算快速处理海量数据。可以简化成四个核心要素：何时触发任务、从哪里读取数据、如
何处理数据、将结果写到哪里。

类似于 MySQL 执行 SQL 查询：你指定从哪个表（数据源） 读取，写 SQL 函数（如 max、sum、group by）做计算，最
后结果展示或写入目标表。

不同的是，Spark 的数据来自分布式存储系统，计算是并行执行，且任务通常由调度系统定时触发。

这样，Spark 让大规模数据处理既灵活又高效。Spark 应用主要由两个核心组件组成：Driver 和 Executor。

Driver 是整个应用的“大脑”，负责接收用户的计算逻辑，构建执行计划（DAG），并调度任务的执行。
它负责协调整个计算过程，管理任务的生命周期。

Executor 是分布在集群各节点上的工作进程，负责执行具体的计算任务，比如读取数据、处理数据、写出结果。
Executor 会将计算结果返回给 Driver 或者写入存储系统。

简单来说，Driver 负责规划和调度，Executor 负责实际干活。
