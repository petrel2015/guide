---
question: "请介绍一下 Redis 中的跳跃表（Skip List）及其应用场景"
tags: [ "Redis", "数据结构", "后端" ]
---

- **核心定义**：跳跃表是一种基于概率维度的多层链表结构。它通过在普通链表之上建立多级索引（索引层），实现了类似二分查找的效率，将查询的时间复杂度从 $O(n)$ 降低到平均 $O(\log n)$。
- **层级结构**：跳跃表由多个层组成，最底层（Level 0）包含所有元素。每个节点随机分配一个高度，并在不同层级拥有指向后续节点的指针。查找时从最高层开始，遇到比目标大的值则下降一层继续搜索。
- **Redis 应用**：跳跃表是 Redis **有序集合（Sorted Set / zset）** 的核心底层结构之一。当 zset 的成员数量较多或成员字符串较长时，Redis 会使用跳跃表（结合哈希表）来存储。
- **设计权衡**：Redis 开发者 antirez 选择跳跃表而非红黑树的主要原因是：**实现更简单**且**范围查询（Range Query）性能更好**。在执行 `ZRANGE` 或 `ZREVRANGE` 等操作时，跳跃表只需在底层链表顺序遍历即可。
- **空间开销**：虽然增加了多级索引指针，但由于每个节点晋升到上一层的概率是固定的（Redis 中为 1/4），其平均空间复杂度仍保持在 $O(n)$，并不会造成显著的内存浪费。
- **并发友好性**：虽然 Redis 核心是单线程，但在多线程并发场景下，跳跃表的修改操作比红黑树更容易通过局部锁实现，避免了复杂的树旋转平衡操作。
