---
question: "Redis如何实现分布式锁"
tags: [ "redis", "分布式", "后端", "锁" ]
---

- **核心指令 SET NX PX**：使用 `SET key value NX PX milliseconds` 命令。`NX` 保证只有 key 不存在时才设置成功（互斥性），`PX` 设置过期时间防止锁死（安全性）。
- **唯一 Value 标识**：Value 必须具有唯一性（如 UUID）。在释放锁时，通过判断 Value 是否匹配来确保“谁加锁谁释放”，防止锁被误删。
- **Lua 脚本原子释放**：释放锁时必须使用 Lua 脚本，将“判断 Value”和“删除 Key”合并为一个原子操作，避免在判断成功后锁正好过期导致误删他人锁。
- **看门狗（Watchdog）机制**：针对业务执行时间超过锁过期时间的情况，需引入定时续期机制（如 Redisson 的 Watchdog），每隔一段时间检查并延长锁的有效期。
- **高可用 Redlock 算法**：在 Redis 集群环境下，通过向过半数（N/2 + 1）个独立的 Redis 节点申请锁来确保容错性，防止单点故障导致锁失效。
- **可重入性实现**：如果需要支持可重入，可以参考 Redisson 实现，利用 Redis 的 Hash 结构存储 `线程标识` 和 `重入计数`，每次重入计数加一。
